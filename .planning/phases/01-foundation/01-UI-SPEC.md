> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

| Property | Value |
|----------|-------|

| Component library | Radix UI (via shadcn) |

| Font | Inter, system-ui, sans-serif |

### shadcn Initialization Command

Run once during Phase 1 task 1 (monorepo scaffold):

```bash
cd packages/client
npx shadcn@latest init --defaults
```

Select when prompted: style = Default, base color = Zinc, CSS variables = yes.

After init, set `darkMode: "class"` in `tailwind.config.ts` and add `class="dark"` to the `<html>` element in `index.html`. This locks the app into dark mode permanently — there is no light mode toggle in v1.

Confirm `packages/client/components.json` exists before proceeding to any UI task.

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, input icon padding, badge inner spacing |
| sm | 8px | Compact element spacing, form field gap, button icon-to-label gap |
| md | 16px | Default element spacing, card padding, form group gap |
| lg | 24px | Section padding, card outer margin, page content top padding |
| xl | 32px | Layout gaps between major page sections |
| 2xl | 48px | Major section breaks, auth card vertical offset from viewport center |
| 3xl | 64px | Page-level vertical spacing on profile page |

Exceptions: Touch targets for OAuth buttons (Google, Discord) must be a minimum of 44px tall to meet WCAG 2.5.5. This overrides any compact spacing applied to those elements.

## Typography

All type uses Inter, falling back to system-ui then sans-serif.
Exactly 4 sizes, exactly 2 weights.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | Form labels, input placeholder, profile stat labels, general copy |
| Label | 14px | 400 (regular) | 1.4 | Helper text, validation messages, secondary profile metadata |
| Heading | 20px | 600 (semibold) | 1.2 | Page titles (Register, Sign In, Profile), card section headers |
| Display | 28px | 600 (semibold) | 1.1 | Username display on profile page, kill count stat value |

No italic weights. No weights other than 400 and 600 in Phase 1.

Number display rule: every game number (kill count, leaderboard rank) rendered in the UI must pass through `formatNumber()` from `packages/shared-types/src/numbers.ts`. Raw integers are never written directly into JSX. This applies to Display-role text and any numeric span in the profile.

## Color

Base palette: shadcn zinc dark. Accent overridden to deep red.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #09090b (zinc-950) | Page background, app shell background |
| Secondary (30%) | #18181b (zinc-900) | Auth card background, profile card background, nav bar background |
| Accent (10%) | #DC2626 (red-600) | Primary submit buttons only (Register button, Sign In button, Connect Google button, Connect Discord button) |
| Destructive | #DC2626 (red-600) | Not used in Phase 1 — no destructive actions exist |
| Surface border | #27272a (zinc-800) | Card borders, input borders, dividers |
| Muted text | #a1a1aa (zinc-400) | Placeholder text, helper text, secondary labels |

Accent reserved for: **primary submit buttons only** — the Register form submit, the Sign In form submit, the "Continue with Google" button, and the "Continue with Discord" button. No other elements use #DC2626 in Phase 1.

CSS variable overrides to add in `packages/client/src/index.css` (inside `.dark` block):

```css

  --primary: 0 72% 51%;           /* hsl equivalent of #DC2626 */
  --primary-foreground: 0 0% 98%;
}
```

## Screen Contracts

### Register Page (`/register`)

Layout: centered single-column, auth card at vertical center of viewport (offset 48px from top on tall screens).

Elements (top to bottom inside card):
1. Heading: "Create account" — 20px, weight 600
2. Username input + label
3. Password input + label (type="password")
4. Submit button: full-width, accent (#DC2626), label "Create account"
5. Separator
6. OAuth section heading: "Or continue with" — 14px, weight 400, zinc-400
7. "Continue with Google" button: variant="outline", full-width, Google icon (lucide not available; use inline SVG or simple text label)
8. "Continue with Discord" button: variant="outline", full-width, Discord icon (same approach)
9. Link to Login: "Already have an account? Sign in" — 14px, zinc-400, "Sign in" underlined

Validation feedback appears as a Label below the relevant input, 14px, #EF4444 (red-500 — slightly lighter than accent for inline error distinction).

### Login Page (`/login`)

Layout: identical structure to Register page.

Elements (top to bottom inside card):
1. Heading: "Sign in" — 20px, weight 600
2. Username input + label
3. Password input + label (type="password")
4. Submit button: full-width, accent (#DC2626), label "Sign in"
5. Separator
6. OAuth section heading: "Or continue with" — 14px, weight 400, zinc-400
7. "Continue with Google" button: variant="outline", full-width
8. "Continue with Discord" button: variant="outline", full-width
9. Link to Register: "No account? Create one" — 14px, zinc-400, "Create one" underlined

### Profile Page (`/profile/:userId`)

Layout: single-column, page-level 64px vertical spacing.

Elements:
1. Username: 28px, weight 600 (Display role)
2. Stats row (horizontal flex, xl/32px gap):
   - "Kills" label (14px, zinc-400) + kill count value (28px, weight 600, white) — value via `formatNumber()`
   - "Rank" label (14px, zinc-400) + rank value (28px, weight 600, white) — rank via `formatNumber()`; shows "—" (em dash) when rank is null (not yet ranked)
3. Separator
4. "Member since" — 14px, zinc-400, date formatted as "March 2026"
5. Sign out link — 14px, zinc-400, no button styling, underlined

OAuth callback (`/auth/google/callback`, `/auth/discord/callback`): no UI rendered. Server issues JWT cookie and immediately redirects to `/profile`. If redirect fails, show plain text "Authentication failed. Return to sign in." with a link back to `/login`.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA — Register | "Create account" |
| Primary CTA — Login | "Sign in" |
| OAuth CTA — Google | "Continue with Google" |
| OAuth CTA — Discord | "Continue with Discord" |
| Register nav link | "Already have an account? Sign in" |
| Login nav link | "No account? Create one" |
| Empty state — Profile kills | Kill count displays "0" via formatNumber(); no special empty state copy needed |
| Empty state — Profile rank | Rank displays "—" (em dash) when kbRank is null |
| Error — Username taken (409) | "That username is already taken. Try a different one." |
| Error — Invalid credentials (401) | "Incorrect username or password." |
| Error — OAuth failure | "Authentication failed. Return to sign in." |
| Error — Network/server (5xx) | "Something went wrong. Please try again." |
| Error — Validation (client-side, empty field) | "This field is required." |
| Error — Validation (password too short) | "Password must be at least 8 characters." |
| Destructive confirmation | Not applicable in Phase 1 — no destructive actions |

Copywriting rules:
- Never expose raw HTTP status codes to the user.
- Never say "Error 409" or "Error 401" in UI copy.
- Keep all error messages under 60 characters where possible.
- Sign-in and sign-out language uses "sign in / sign out" (not "log in / log out") throughout for consistency.

## Interaction States

Every interactive element must implement all four states. No exceptions.

| State | Button (default/accent) | Button (outline/OAuth) | Input |
|-------|------------------------|----------------------|-------|
| Default | bg-primary (#DC2626), white text | transparent bg, zinc-800 border | zinc-950 bg, zinc-800 border |
| Hover | bg-primary/90 (10% opacity reduction) | bg-zinc-800, white text | zinc-800 bg, zinc-600 border |
| Focus | ring-2 ring-primary ring-offset-2 | ring-2 ring-zinc-400 ring-offset-2 | ring-2 ring-primary ring-offset-2 |
| Disabled | opacity-50, cursor-not-allowed | opacity-50, cursor-not-allowed | not applicable in Phase 1 |
| Loading | Show spinner (lucide `Loader2` icon, animate-spin), disable pointer events | not applicable | not applicable |

Loading state applies to all four submit/CTA buttons while the async request is in flight. Button text is replaced by a spinner. Width does not change (use a fixed-width container or `min-w`).

## Accessibility Contract

- All inputs have an associated `<Label>` linked via `htmlFor` / `id`.
- Color contrast: zinc-50 (#fafafa) on zinc-950 (#09090b) = 19.6:1 (AAA). Zinc-400 on zinc-950 = 4.8:1 (AA). Accent (#DC2626) on zinc-950 = 4.5:1 (AA minimum met for large text; primary buttons use large text).
- OAuth buttons are minimum 44px tall (WCAG 2.5.5).
- Form submit buttons are `type="submit"` so Enter key triggers submission.
- Error messages are associated with inputs via `aria-describedby`.
- No color-only error indicators — validation errors always include text copy alongside any red color.
- Focus ring is always visible (ring-2, never outline: none without replacement).
