## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline label gaps |
| sm | 8px | Compact padding (gap-2), stat row gaps |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding within game layout |
| xl | 32px | Layout column gaps (boss area / sidebar) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Touch targets for the attack button: minimum 44px height (`min-h-[44px]`), consistent with Phase 1 OAuth buttons вҖ” source: Login.tsx
- Boss HP bar height: 16px (`h-4`) вҖ” distinct from spacing scale, this is a component dimension
- Floating damage number absolute positioning: `y: -60px` animation travel distance вҖ” source: RESEARCH.md Pattern 5

## Typography

| Role | Size | Weight | Line Height | Tailwind Class | Usage |
|------|------|--------|-------------|----------------|-------|
| Body | 14px | 400 | 1.5 | `text-sm` | Player names in sidebar, muted stat labels, error messages |
| Label | 16px | 400 | 1.5 | `text-base` | Standard body copy, boss lore snippet |
| Heading | 20px | 600 | 1.2 | `text-xl font-semibold` | Boss name heading, section headers |

Source: Profile.tsx and Login.tsx establish `text-xl font-semibold` for headings and `text-[28px] font-semibold leading-[1.1]` for stat display вҖ” carried forward exactly.

**Floating damage number exception:**
- Size: 20px (`text-xl`)
- Weight: 700 (`font-bold`)
- Color: `text-yellow-400` (not part of the 60/30/10 color contract вҖ” this is game feedback only)
- Source: RESEARCH.md Pattern 5 вҖ” `className="... text-yellow-400 font-bold text-xl"`

## Color

| Role | oklch Value | Hex Equivalent | Usage |
|------|-------------|----------------|-------|
| Dominant (60%) | `oklch(0.141 0.005 285.823)` | ~`#1c1c23` (zinc-950 range) | Page background, full-bleed game canvas background |
| Secondary (30%) | `oklch(0.178 0.006 285.886)` | ~`#27272f` (zinc-900 range) | Cards (boss card, player sidebar card), nav/header surface |
| Accent (10%) | `oklch(0.497 0.209 27.157)` | ~`#DC2626` (red-600) | Reserved: attack button fill, boss HP bar fill, ring/focus outline on active inputs |
| Muted text | `oklch(0.705 0.015 286.067)` | ~`#a1a1aa` (zinc-400) | Inactive player names, stat labels, timestamps |
| Destructive | `oklch(0.704 0.191 22.216)` | ~`#f87171` (red-400 tone) | Field validation errors only вҖ” no destructive actions in Phase 2 |
| Damage number | `text-yellow-400` (Tailwind) | `#facc15` | Floating damage feedback only вҖ” not part of surface 60/30/10 |

Accent reserved for:
1. Attack button background fill (primary `<Button>` variant)
2. Boss HP bar fill color (the Progress indicator track)
3. Focus ring (`ring-primary`) on the attack button when keyboard-focused
4. The "Killing Blow" label/badge if shown in Phase 2 (placeholder for Phase 4 announcement)

Accent is NOT used on: player sidebar rows, boss name, boss lore text, HP numbers, damage contribution figures.

Source: index.css `--primary: oklch(0.497 0.209 27.157)` and `.dark { --primary: 0 72% 51%; }` вҖ” confirmed red-600 decision from STATE.md Phase 01-foundation decision.

## Component Inventory

New components for Phase 2 (all in `packages/client/src/components/`):

| Component | File | Key Props | Behavior |
|-----------|------|-----------|----------|
| BossHpBar | `BossHpBar.tsx` | `hp: number`, `maxHp: number` | shadcn `<Progress value={(hp/maxHp)*100}>`; red-600 fill; 16px height; smooth CSS transition on `value` change via `transition-all duration-150` |
| BossSprite | `BossSprite.tsx` | `bossNumber: number`, `isDefeated: boolean` | Placeholder `<div>` with boss number; `motion.div` with `animate={{ opacity: isDefeated ? 0 : 1, scale: isDefeated ? 1.2 : 1 }} transition={{ duration: 0.4 }}` death animation |
| DamageNumbers | `DamageNumbers.tsx` | none (subscribes to socket internally) | `AnimatePresence` list of `motion.div`; `initial={{ opacity: 1, y: 0 }}`, `animate={{ opacity: 0, y: -60 }}`, `transition={{ duration: 1.0, ease: 'easeOut' }}`; auto-remove after 1200ms |
| PlayerSidebar | `PlayerSidebar.tsx` | `players: ActivePlayer[]` | Scrollable list; each row: username (text-sm) + formatted damage contribution (text-sm font-semibold); sorted by damage descending |

New page: `packages/client/src/pages/Game.tsx`
- Route: `/game` (authenticated, guarded with same `useSessionStore` pattern as Profile)
- Mounts socket subscription via `subscribeToGame()` on mount, unsubscribes on unmount
- Fetches `GET /boss/current` on mount to hydrate `bossStore` before first WebSocket event

New stores (in `packages/client/src/stores/`):
- `bossStore.ts` вҖ” `bossId`, `name`, `hp`, `maxHp`, `bossNumber`, `isDefeated`
- `playerStore.ts` вҖ” `activePlayers: ActivePlayer[]`

## Animation Contract

| Animation | Component | Library | Duration | Easing |
|-----------|-----------|---------|----------|--------|
| Floating damage number float-up | DamageNumbers | motion | 1000ms | easeOut |
| Floating damage number fade out | DamageNumbers | motion | 1000ms | easeOut (same transition) |

## Interaction States

### Attack Button
| State | Visual |
|-------|--------|
| Default | `bg-primary text-primary-foreground` (red-600 fill, white text) |
| Hover | `hover:bg-primary/90` (10% opacity reduction) |
| Focus | `focus:ring-2 focus:ring-primary focus:ring-offset-2` |
| Disabled (boss defeated, awaiting respawn) | `disabled:opacity-50 disabled:cursor-not-allowed` |

### Boss HP Bar
| State | Visual |
|-------|--------|
| Normal (>10% HP) | red-600 fill on zinc-800 track |
| Low HP (<10%) | same red-600 fill вҖ” no color change in Phase 2 |
| Boss defeated | `value=0`; immediately transitions on `boss:death` |

### Player Sidebar
| State | Visual |
|-------|--------|
| Active players present | Sorted list of username + damage rows |
| No players yet (initial load) | Show "Waiting for players..." in `text-sm text-muted-foreground` |

### Loading State (Game.tsx initial load)
- Show nothing until both `GET /boss/current` resolves and `bossStore.bossId` is populated
- Do not show a skeleton or spinner вҖ” return `null` (consistent with Phase 1 `Profile.tsx` pattern)

## Copywriting Contract

| Element | Copy | Notes |
|---------|------|-------|
| Attack button | "Attack" | Single word, imperative. No icon in Phase 2. |
| Boss HP display | "{hp} / {maxHp}" | Both values formatted with `formatNumber()` from shared-types |
| Player sidebar heading | "In the Fight" | 14px, weight 600, zinc-50 |
| Sidebar player row вҖ” damage | "{damage} dmg" | `formatNumber()` applied to damage value |
| Boss defeated overlay | "Defeated вҖ” new boss spawning..." | Shown while `isDefeated === true` and before `boss:spawn` arrives; `text-sm text-muted-foreground` centered below sprite |
| No players in sidebar | "Waiting for players..." | `text-sm text-muted-foreground` |
| Connection error | "Connection lost. Reconnecting..." | `text-sm text-destructive` shown when socket disconnects |
| Boss name | Dynamically set from server `boss.name` field | Displayed as `text-xl font-semibold` heading |

No destructive actions in Phase 2. No confirmation dialogs required.

## Accessibility Contract

- Attack button: `aria-label="Attack the boss"` when icon-only; full text label used, so no aria-label needed
- HP bar: `role="progressbar"`, `aria-valuenow={hp}`, `aria-valuemin={0}`, `aria-valuemax={maxHp}`, `aria-label="Boss HP"` вҖ” shadcn `<Progress>` provides these natively
- Damage numbers: `aria-hidden="true"` on the entire `DamageNumbers` container вҖ” purely decorative, not read by screen readers
- Boss sprite: `aria-hidden="true"` вҖ” decorative placeholder
- Player sidebar list: `<ul>` with `<li>` items; each `<li>` announces "username: damage dmg" as text
