## Color

All values are from `packages/client/src/index.css` CSS custom properties.

| Role | CSS Variable | Approximate Value | Usage |
|------|-------------|------------------|-------|
| Dominant (60%) | `--background` | oklch(0.141 0.005 285.823) — near-black zinc | Page background, full-screen base |
| Secondary (30%) | `--card` | oklch(0.178 0.006 285.886) — dark zinc | UpgradePanel card, gold display card, any new panel surfaces |
| Accent (10%) | `--primary` | oklch(0.497 0.209 27.157) — red-600 (#DC2626) | See reserved list below |
| Muted text | `--muted-foreground` | oklch(0.705 0.015 286.067) — zinc-400 | Upgrade cost labels when insufficient gold, offline reward secondary text |
| Destructive | `--destructive` | oklch(0.704 0.191 22.216) — red-400 | No destructive actions in this phase — not used |

Accent (`--primary` / red-600) reserved exclusively for:
1. The Attack Boss button (already established in Phase 2)
2. The Boss HP bar fill (already established in Phase 2)
3. Active-tab DPS bonus indicator badge (one small visual cue that the tab is active)
4. Upgrade button when the player can afford the upgrade (affordability call-to-action)

Accent is NOT used for: gold numbers, upgrade level counters, stat labels, sidebar player list, offline reward notification background.

Gold numbers use `text-yellow-400` — consistent with existing DamageNumbers.tsx color. This is not a new color token; it is the Tailwind utility already in use.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (upgrade) | "Upgrade" — used on each stat's upgrade button |
| Gold display label | "Gold" |
| ATK stat label | "ATK" with subtext "Flat damage per hit" |
| CRIT stat label | "CRIT" with subtext "Critical hit chance" |
| SPD stat label | "SPD" with subtext "Attacks per second" |
| Upgrade button affordability subtext | "Cost: {formatNumber(cost)} gold" |
| Upgrade panel heading | "Upgrades" |
| Empty gold state (0 gold, cannot afford anything) | Heading: "Attack to earn gold" / Body: "Every hit earns gold. Spend gold to grow stronger." |
| Insufficient gold state (can see cost but cannot afford) | Button disabled. No modal. Tooltip or muted label: "Not enough gold" |
| Offline reward notification heading | "Welcome back!" |
| Offline reward notification body | "You earned {formatNumber(gold)} gold while away ({hours}h {minutes}m offline)." |
| Error state (upgrade HTTP 400) | "Upgrade failed — not enough gold." shown inline below the upgrade button, auto-dismisses after 3s |
| Error state (upgrade HTTP 500) | "Something went wrong. Try again." shown inline, auto-dismisses after 5s |
| Active tab indicator | "Active bonus: 2x DPS" — small badge or inline label near the gold display, only visible when tab is active |

Destructive actions in Phase 3: None. No confirmation dialogs required.

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Upgrade button click (affordable) | Optimistic UI: increment level display immediately, deduct cost from local gold balance. Server confirms via `player:stats_update`. If server rejects, revert. |
| Upgrade button click (insufficient gold) | Button is disabled (`disabled` attribute). No click event fires. Muted cost label indicates shortfall. |
| Gold balance update | Animated number increment using `formatNumber`. No full re-render flash. Updates via `player:gold_update` socket event consumed by `progressionStore`. |
| Offline reward toast | Appears automatically on reconnect if `player:offline_reward` event received. Auto-dismisses after 8 seconds. No manual dismiss required but user can tap/click to dismiss early. |
| Tab active/inactive | Client fires `player:heartbeat` every 5 seconds via `visibilitychange` + `setInterval`. Active bonus badge appears when heartbeat is live (local state in `progressionStore.isTabActive`). |
| Upgrade level display | Shows current level as integer (e.g. "Lv. 3"). Updates immediately on optimistic update. |

## Layout

Phase 3 extends the existing Game.tsx two-column layout:

```
┌─────────────────────────────────────┬────────────┬────────────┐
│           Boss Area (flex-1)        │  Upgrade   │  Player    │
│  [Boss Name]                        │  Panel     │  Sidebar   │
│  [Boss Sprite]                      │  (w-64)    │  (w-64)    │
│  [HP Bar]                           │            │            │
│  [Attack Button]                    │  Gold: XXX │            │
│                                     │  ATK Lv.N  │            │
│                                     │  CRIT Lv.N │            │
│                                     │  SPD Lv.N  │            │
└─────────────────────────────────────┴────────────┴────────────┘
```

- UpgradePanel is positioned between the boss area and PlayerSidebar: `flex-shrink-0 w-64`
- On narrower viewports (< 1024px), UpgradePanel stacks below boss area (flex-col wrap)
- PlayerSidebar remains at far right as in Phase 2

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Upgrade button disabled state | Use `disabled` attribute — native button semantics, not `aria-disabled` |
| Gold balance live region | `aria-live="polite"` on the gold balance span so screen readers announce updates without interrupting |
| Upgrade cost | Visible text label — no icon-only affordance |
| Offline reward toast | `role="status"` with `aria-live="polite"` |
| Active bonus badge | `aria-label="Active DPS bonus active"` — decorative text gets a meaningful label |
