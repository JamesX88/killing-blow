# Phase 5 вҖ” UI Design Contract

> Visual and interaction contract for Phase 5: UI Overhaul вҖ” Mobile Game Feel.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design Objective

The current UI is a collection of blocks on a page. This phase transforms it into a full-viewport immersive game. Visual reference: Tap Titans / Clicker Heroes idle RPG aesthetic. Art direction: dark gothic dungeon, atmospheric glow, every action must feel impactful.

The single most important change: the game viewport fills the screen. The boss occupies center stage. HP is dramatic. Panels live on the periphery. The layout communicates "you are inside a dungeon" not "you are on a dashboard".

## Boss Sprite Contract

The current `BossSprite` renders a circle with `#bossNumber`. This phase replaces it with a proper visual centerpiece.

### Option A вҖ” Placeholder Boss Art (no asset pipeline needed)

Use SVG-based or CSS-art boss silhouettes keyed to `bossNumber % 5` for visual variety:
- 5 distinct silhouette shapes (e.g. dragon, golem, wraith, spider, lich)
- Rendered as large SVG icons from lucide-react or custom inline SVGs
- Minimum rendered size: 200x200px on mobile, 280x280px on desktop
- Apply red-tinted `filter: drop-shadow(0 0 16px rgba(220,38,38,0.5))` to all silhouettes
- Idle animation: slow breathing scale cycle `scale(1) вҶ’ scale(1.02) вҶ’ scale(1)`, 3s ease-in-out infinite
- Hit animation: `scale(0.95)` flash on `boss:damage_dealt` event, 100ms ease-in-out (use CSS class toggle via React state)
- Death animation: scale up + fade to red, 400ms ease-out (existing `isDefeated` logic extended)

### Option B вҖ” Image sprites (requires asset pipeline)

If assets are provided: render via `<img>` with `object-fit: contain`, constrained to 200-300px. Asset pipeline is out of scope for this phase. Use Option A.

**Final decision: Use Option A.** Boss art is CSS/SVG placeholder art keyed to `bossNumber % 5`. No image asset pipeline needed.

The current HP bar is `height: 16px`, flat, no glow. This phase makes it dramatic.

| Property | Current | Phase 5 Contract |

| Height | 16px | 28px |

## Screen Effects Contract

New effects, none currently exist:

### Kill Flash (boss death)

- Trigger: `boss:death` socket event
- Effect: full-screen overlay div, `position: fixed; inset: 0; pointer-events: none; z-index: 40`
- Color: `rgba(220, 38, 38, 0.25)` (red-600 at 25% opacity)
- Animation: `@keyframes kill-flash` вҖ” opacity 0 вҶ’ 1 вҶ’ 0, total duration 600ms
- Implementation: React state boolean `killFlashActive`, set true on `boss:death`, auto-reset after 600ms

### Screen Shake (boss death)

- Trigger: same `boss:death` event as kill flash
- Effect: CSS class `.screen-shake` applied to the game root container for 400ms
- Animation: `@keyframes screen-shake` вҖ” translate-based micro-jitter (max Вұ3px), 400ms
- Implementation: add/remove class on game root div via ref or state
- Accessibility: wrap in `@media (prefers-reduced-motion: no-preference)` вҖ” skip shake if user prefers reduced motion

### Hit Feedback (per attack)

- Trigger: `boss:damage_dealt` socket event (player's own hit)
- Effect: boss sprite container flashes `brightness(1.3)` for 80ms via CSS `filter`
- No screen shake on individual hits вҖ” only on kill
- Implementation: CSS class toggle `.boss-hit-flash { animation: boss-flash 80ms ease-in-out; }`

## Interaction Contracts

### Attack Button

- Minimum height: 64px (primary action; larger than standard 44px)
- Width: full width on mobile; constrained max-width 320px centered on desktop
- Idle state: `bg-primary` (red-600), bold label "Attack"
- Disabled state (boss defeated): `opacity-50`, no animation
- Press feedback: `active:scale-95` CSS transform, 100ms
- No long-press or hold behavior вҖ” single tap triggers one emit (auto-attack loop handles repeat)

### Mobile Upgrade/Title Drawer

- Trigger: persistent bottom-docked tab bar with two tabs: "Upgrades" | "Titles"
- State: collapsed by default (only tab labels visible, 48px height); tap to expand to 60vh slide-up panel
- Animation: `translateY(100%) вҶ’ translateY(0)`, 250ms ease-out spring
- Close: tap tab again or tap backdrop (above drawer)
- z-index: 20 (above game content, below announcements z-50)

### Killing Blow Announcement (enhanced)

Inherits Phase 4 contract. Phase 5 additions:
- Entry animation: spring physics (`type: "spring", stiffness: 400, damping: 20`) replaces linear ease
- Card receives `box-shadow: 0 0 40px rgba(220,38,38,0.4)` glow on entry
- Backdrop changes from `bg-black/70` to `bg-black/80` for more drama
- Winner name size: 28px bold (upgraded from 20px semibold in Phase 4 spec)
- "Killing Blow!" headline: 36px bold (upgraded from 28px вҖ” this is the display moment)

### Navigation

- Leaderboard link: moves from plain `<a>` text link to a small icon button (trophy icon from lucide-react) in a fixed top-right position over the game viewport
- Back-to-game from Leaderboard: same small icon/text link, top-left

## Accessibility Contract

Phase 5 introduces significant motion. Motion-sensitive users must be protected:

- All `@keyframes` (screen-shake, kill-flash, hp-pulse, boss-hit-flash) are wrapped in `@media (prefers-reduced-motion: no-preference)` вҖ” if user prefers reduced motion, these animations do not run
- Damage numbers: still render and float, but without scale pop-in if reduced motion is preferred
- Boss sprite idle breathe: disabled under prefers-reduced-motion
- Framer Motion components: use `useReducedMotion()` hook to skip or simplify animations
- Attack button: `aria-label="Attack the boss"` (already in codebase вҖ” carry forward)
- Kill flash overlay: `aria-hidden="true"` вҖ” purely decorative
- DungeonBackground: `aria-hidden="true"` вҖ” purely decorative
